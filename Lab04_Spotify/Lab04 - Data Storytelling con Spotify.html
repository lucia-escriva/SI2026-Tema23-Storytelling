El escenario

Eres analista de datos en SurLabel, una discográfica independiente con sede en Madrid. Tu directora artística acaba de entrar en tu despacho:

"Necesito entender cómo funciona el éxito en Spotify España. Tenemos tres lanzamientos previstos para el próximo trimestre y no sé si estamos planificando bien las campañas. Dame algo que pueda enseñarle al equipo el lunes."

Tienes a tu disposición los rankings diarios de Spotify España de 2025: más de 200 días de datos, con las posiciones del top 50 y los streams reales de cada canción.

Tu trabajo no es hacer un análisis exploratorio completo. Es construir una historia que ayude a tomar una decisión concreta.

Marco teórico

Antes de abrir los datos, interioriza los tres ejes del análisis explicativo según Storytelling with Data (Knaflic, 2015):

  1. Contexto primero — El análisis empieza por la audiencia, no por los datos. ¿Quién escucha esto, qué decisión tiene que tomar, qué tono corresponde?
  2. El gráfico es el argumento — No ilustras una conclusión; el visual es la conclusión. El título debe decir qué concluir, no qué ver.
  3. La atención es escasa, el ruido es gratis — Todo elemento visual que no contribuye a la historia resta. Eliminar es diseñar.

Cada caso activa un subconjunto específico de estas herramientas. Lee los tres antes de elegir el tuyo.

A

El Reloj de Arena

¿Cuánto tiempo tienes para aprovechar un hit?

La pregunta de negocio

La directora quiere saber cuánto dura el momento de máxima atención sobre una canción. Tiene dos lanzamientos simultáneos previstos y no sabe si tiene recursos para dar soporte a ambos al mismo tiempo o si debería escalonarlos.

Big Idea · escríbela antes de tocar los datos

"Un hit en Spotify España pierde la mitad de sus streams en menos de [X] días: hay una ventana corta y predecible para capitalizar el lanzamiento."

El análisis debe confirmar o refutar esa hipótesis, y concretar el valor de X.

Paso 1 — Construye la curva de vida de un hit

Filtra canciones que hayan llegado al top 10 en algún momento y normaliza sus streams respecto al pico máximo:

Pythontop10_songs = df[df['rank'] <= 10][['track_name','artist_names']].drop_duplicates()
merged = df.merge(top10_songs, on=['track_name','artist_names'])
merged['streams_norm'] = merged.groupby(['track_name','artist_names'])['streams'] \

B

Boom o Slow Burner

Naces hit o te conviertes en uno: dos estrategias de lanzamiento

La pregunta de negocio

La directora tiene un artista nuevo sin base de fans consolidada y uno ya establecido con comunidad leal. ¿Deberían lanzar sus singles de la misma manera? Los datos sugieren que no: existen dos patrones de éxito radicalmente distintos, y cada uno requiere una estrategia diferente.

Big Idea · escríbela antes de tocar los datos

"En Spotify España conviven dos tipos de hit: los que arrasan en el debut y se desinflan rápido (Boom), y los que crecen en silencio hasta convertirse en referentes (Slow Burner). Confundir los dos es tirar el presupuesto."

Teoría SWD aplicada

PrincipioCómo se activa en este caso
Contexto y audienciaLa directora no sabe qué es un "percentil de streams"; necesita una metáfora, no una estadística
Visual adecuadoDos curvas medias superpuestas sobre fondo de ruido: comparación directa entre patrones
Atributos preatentivos: colorRojo para Boom (urgencia), azul para Slow Burner (constancia); gris para el resto
Atributos preatentivos: posiciónPequeños múltiplos si la superposición genera confusión
Elimina el clutterSin leyenda lateral; etiqueta directamente las dos curvas en el gráfico

Guía de análisis

Paso 1 — Clasifica cada canción

Una canción es Boom si más del 50 % de sus streams totales ocurren en los primeros 7 días. Es Slow Burner si tarda más de 7 días en alcanzar el top 10.

Rdebut_info <- df |>
  dplyr::group_by(track_name, artist_names) |>
  dplyr::arrange(fecha) |>
  dplyr::summarise(
    debut_rank    = dplyr::first(rank),
    debut_date    = min(fecha),
    total_streams = sum(streams),
    streams_7d    = sum(streams[days_on_chart <= 7]),
    first_top10   = min(fecha[rank <= 10], na.rm = TRUE),
    .groups = "drop"
  ) |>
  dplyr::mutate(
    prop_early    = streams_7d / total_streams,
    days_to_top10 = as.numeric(first_top10 - debut_date),
    hit_type = dplyr::case_when(
      prop_early >= 0.5  ~ "Boom",
      days_to_top10 > 7  ~ "Slow Burner",
      TRUE               ~ "Otros"
    )
  )
Pythondef classify(g):
    total    = g['streams'].sum()
    early    = g[g['days_on_chart'] <= 7]['streams'].sum()
    top10    = g[g['rank'] <= 10]['fecha'].min()
    debut_dt = g['fecha'].min()
    d2top10  = (top10 - debut_dt).days if pd.notna(top10) else 999
    prop     = early / total if total > 0 else 0
    if prop >= 0.5:      return 'Boom'
    if d2top10 > 7:      return 'Slow Burner'
    return 'Otros'

hit_types = (df.sort_values('fecha')
               .groupby(['track_name','artist_names'])
               .apply(classify)
               .reset_index(name='hit_type'))
df = df.merge(hit_types, on=['track_name','artist_names'])

C

Los Inmortales

Cuando una canción deja de ser un hit y se convierte en un himno

La pregunta de negocio

La directora está pensando en la estrategia de catálogo a largo plazo. Le han dicho que "Soldadito Marinero" de Fito y los Fitipaldis lleva más de 6 años entre las canciones más escuchadas de España. No lo termina de creer. ¿Es un caso aislado o hay un patrón?

Big Idea · escríbela antes de tocar los datos

"Un 3% de las canciones del top 50 español llevan más de 4 años en el ranking. No son hits: son activos permanentes. Y el catálogo clásico puede valer más que el próximo lanzamiento viral."

Teoría SWD aplicada

PrincipioCómo se activa en este caso
Comprende el contextoAudiencia no técnica: necesita sentir la magnitud del outlier, no verlo en una tabla

Atributos preatentivos: colorLa masa gris hace que los outliers coloreados "salten" sin señalización extra
Anotación directaEtiqueta los outliers con nombre + años en chart: ese dato es la historia
El título es la conclusiónEl título no describe el gráfico; resume el insight que la directora se lleva

Guía de análisis

Paso 1 — Calcula la longevidad máxima de cada canción

Rlongevity <- df |>
  dplyr::group_by(track_name, artist_names) |>
  dplyr::summarise(
    max_days     = max(days_on_chart, na.rm = TRUE),
    mean_streams = mean(streams,      na.rm = TRUE),
    avg_rank     = mean(rank,         na.rm = TRUE),
    .groups = "drop"
  ) |>
  dplyr::mutate(
    years_on_chart = max_days / 365,
    categoria = dplyr::case_when(
      max_days >= 1825 ~ "Eterno (5+ años)",
      max_days >= 1095 ~ "Resistente (3–5 años)",
      max_days >= 365  ~ "Longevo (1–3 años)",
      TRUE             ~ "Efímero (< 1 año)"
    )
  )
Pythonlongevity = df.groupby(['track_name','artist_names']).agg(
    max_days     = ('days_on_chart', 'max'),
    mean_streams = ('streams',       'mean'),
    avg_rank     = ('rank',          'mean')
).reset_index()

longevity['years_on_chart'] = longevity['max_days'] / 365

def categorize(d):
    if d >= 1825: return 'Eterno (5+ años)'
    if d >= 1095: return 'Resistente (3–5 años)'
    if d >= 365:  return 'Longevo (1–3 años)'
    return 'Efímero (< 1 año)'

longevity['categoria'] = longevity['max_days'].apply(categorize)

Nota metodológica — incluir en el entregable

Los streams del dataset corresponden sólo a los días en que la canción estuvo en el top 50. El valor real del catálogo es mayor porque una canción puede seguir generando streams fuera del ranking. Señala esta limitación explícitamente en tu presentación.
